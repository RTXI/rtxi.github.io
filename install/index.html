Installing RTXI

There are three ways to install RTXI 2.0:

Standard
Use our Debian-based live CDs. It's easier than manually installing RTXI, but differences between your system and the one used to make the live CD could affect real-time performance.

Custom
Install a generic Linux image, compile a Xenomai-patched kernel, and then install RTXI from source. It's more difficult, but the end result is better optimized. We recommend this option.

Upgrade
If you already use a previous version of RTXI, you can install the RTXI GUI without recompiling a kernel. You'll lose out on the improvements in the current kernel, though.

{% increment standardcount %}. Download the live RTXI 2.0 image.

We currently have Ubuntu and Debian live CDs. You can use them to test RTXI without touching your hard drive, format your hard drive, and install RTXI alongside existing operating systems.

 

Our legacy version, RTXI 1.4, is available for now. Keep in mind that v1.4 runs on the no-longer-maintained Ubuntu 10.04, and it requires now-deprecated libraries.

 

{% increment standardcount %}. Burn the image to a DVD or a USB drive.

You can use whatever DVD burner is available on your operating system. If you want a live USB, you may need additional software. For different operating systems, check out:

Linux
Clone the ISO image from the command line with dd or use applications like UNetbootin or USB Creator. Check your distro's package repository to see if they're available.

Windows
Try LinuxLive USB Creator or UNetbootin.

MacOS
Same programs. Try LinuxLive USB Creator or UNetbootin.

We cannot not guarantee these installation programs will produce bootable live USBs for your system. This area tends to be a bit finnicky. Thus far, we've had greater success with burning DVDs than making USBs. Should you have any difficulties, let us know.

 

{% increment standardcount %}. Boot from the live image.

If your computer isn't already set to boot from USB or DVD, you'll need to enter your BIOS settings. When cold-booting your computer, hit the necessary keys, usually F2 or F4, and open the menu. (If you miss the timing, restart your computer and try again.)

From the menu, look for your boot options. There you can manually override the boot order of devices on your system. If you're booting from a DVD or USB, look for your DVD drive or USB label, respectively, and set them to be first in your boot order. Save you changes and exit your BIOS menu.

Reminder: In the next step, you will be installing an operating system on your hard drive. Back it up if you haven't already.

 

{% increment standardcount %}. Install RTXI via the installer.

When booting, you can choose to go directly to the installer or boot into the live desktop. Currently, the installer does not work from within the desktop, so to install RTXI, go to either install option in the boot menu.

Note: If you do go into the live desktop, you can try RTXI by selecting it from the applications menu or by opening a terminal and running sudo rtxi. If you want to install RTXI afterwards, simply restart the machine and select either installer from the boot menu.

The installer will guide you through setting up the system name, passwords, networks, and partitioning. If unsure of what to do, use defaults when available.

 

{% increment standardcount %}. Reboot into RTXI.

When the installer is done, you will be prompted to reboot your system. Reboot (and be careful to not re-enter the live image). RTXI is now installed!

 

{% increment standardcount %}. Give yourself sudoer privileges.

RTXI requires root privileges. To do this, you can either use su to become the root user, or you can use sudo to run RTXI. For the latter option, you'll need to add yourself to the sudoers group. In the command line, run:

{% highlight bash %} $ su # usermod -aG sudo USERNAME # exit {% endhighlight %}

The su command will prompt you for the root password for your system. Enter it, and you will drop into a root shell. From there, execute the usermod command given above, where USERNAME is substituted with your username. Once done, log out of the system and then log back in. You now have sudo privileges, and you can run RTXI by clicking the icon in the applications menu or by entering sudo rtxi in the terminal.

Note: You may need to install video drivers, but at this point, what do you is for your specific purposes. Our modules are listed on our Modules page.

{% increment upgradecount %}. Clone the RTXI GitHub repository.

If you haven't already a clone of our GitHub repo, run the following in the terminal. RTXI 2.0 is currently on the qt4 branch.

{% highlight bash %} $ git clone https://github.com/rtxi/rtxi.git $ cd rtxi $ git checkout qt4 {% endhighlight %}

If you already have a clone of the repository, just make sure you switch to the qt4 branch and pull any updates that may have taken place.

{% highlight bash %} $ git checkout qt4 $ git pull origin master {% endhighlight %}
 

{% increment upgradecount %}. Install and/or update dependencies.

Run the install_dependencies.sh script to download dependencies. The script is maintained for Ubuntu 12.04, 14.04, and Debian 7. If you are using a different or outdated distribution, like Ubuntu 10.04, you will have to look for differently-named packages. For old operating systems, some of these packages may no longer exist, and if that's the case, we strongly encourage you to upgrade to Debian.

{% highlight bash %} $ cd scripts/ $ sudo ./install_dependencies.sh {% endhighlight %}
 

{% increment upgradecount %}. Install and/or update dependencies.

Run the installation script, also in the scripts/ directory.

{% highlight bash %} $ sudo ./install_rtxi.sh {% endhighlight %}

You will be prompted to select a configuration: 1 for Xenomai (real-time) or 2 for POSIX (non-real-time). Soon, RTXI will be compiled and installed. When finished, you can run v2.0 with:

{% highlight bash %} $ sudo rtxi {% endhighlight %}
 

{% increment upgradecount %}. Install modules.

All available modules in 1.4 came packaged with RTXI. For 2.0, we have split modules into separate GitHub repositories so that users can install only what they need and also so that they can fork individual modules for customization, rather than having to fork all of RTXI.

What this means for you is that at this point, you've updated RTXI to v2.0, but your modules are still configured for v1.4. You will have to update them manually. Go to our modules page to find all the modules upgraded for 2.0 you need. If you're using your own custom modules, then they will need to be updated to use Qt4 instead of Qt3. Check out the Qt documentation for porting to Qt4 for help.
