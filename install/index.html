Installing RTXI

There are three ways to install RTXI 2.0:

Standard
Use our Debian-based live CDs. It's easier than manually installing RTXI, but differences between your system and the one used to make the live CD could affect real-time performance.

Custom
Install a generic Linux image, compile a Xenomai-patched kernel, and then install RTXI from source. It's more difficult, but the end result is better optimized. We recommend this option.

Upgrade
If you already use a previous version of RTXI, you can install the RTXI GUI without recompiling a kernel. You'll lose out on the improvements in the current kernel, though.

1. Download the live RTXI 2.0 image.

 

2. Burn the image to a DVD or a USB drive.

You can use whatever DVD burner is available on your operating system. If you want a live USB, you may need additional software. For different operating systems, check out:

Linux
Clone from the command line with dd or use applications like UNetbootin or USB Creator. Check your distro's package repository.
Windows
Try LinuxLive USB Creator or UNetbootin
MacOS
Same programs. Try LinuxLive USB Creator or UNetbootin

We cannot not guarantee these installation programs will produce bootable live USBs for your system. This area tends to be a bit finnicky. Thus far, we've had greater success with burning DVDs than making USBs. Should you have any difficulties, let us know.

3. Boot from the live image.

If your computer isn't already set to boot from USB or DVD, you'll need to enter your BIOS settings. When cold-booting your computer, hit the necessary keys, usually F2 or F4, and open the menu. (If you miss, restart your computer and try again.)

From the menu, look for your boot options. There you can manually override the boot order of devices on your system. If you're booting from a DVD or USB, look for your DVD drive or USBlabel, respectively.

Reminder: In the next step, you will be installing an operating system on a hard drive. Make sure there isn't anything on it you need, like data or another operating system. If there is, back it up somewhere or get an empty one.

4. Install RTXI via the installer.

When booting, you can choose to go directy to the install or boot into a live desktop. If you choose the latter option, you can open the installer from the Applications menu. (If you want to try a non-real-time version of RTXI, open a terminal and run n-rtxi.)

Start the installer. You will be guided through setting up the system name, passwords, networks, and partitioning. If unsure, use defaults when available.

5. Reboot into RTXI.

When the installer is done, you will be prompted to reboot your system. Reboot (and be careful to not re-enter the live image). RTXI is now installed!

You can now start RTXI from the command line. Open a terminal and run sudo rtxi. You may need to install video drivers, but at this point, the next steps are specific to your system.

1. Download a generic Debian installation image.

 

2. Burn the image to a DVD or a USB drive.

3. Follow the built-in installer and install the operating system.

4. Patch and install a real-time kernel. Go to our GitHub wiki for detailed instructions.

5. Clone our GitHub repository and install RTXI. Check out our Github wiki for instructions.

1. Clone the RTXI GitHub repository.

If you haven't already a clone of our GitHub repo, run the following in the terminal. RTXI 2.0 is currently on the qt4 branch. 

$ git clone https://github.com/rtxi/rtxi.git
$ cd rtxi
$ git checkout qt4

If you already have a clone of the repository, just make sure you switch to the qt4 branch and pull any updates that may have taken place. 

$ git checkout qt4
$ git pull origin master

2. Install and/or update dependencies.

Run the install_dependencies.sh script to download dependencies. The script is maintained for Ubuntu 12.04, 14.04, and Debian 7. If you are using a different or outdated distribution, like Ubuntu 10.04, you will have to look for differently-named packages. For old operating systems, some of these packages may no longer exist, and if that's the case, we strongly encourage you to upgrade to Debian. 

$ cd scripts/
$ sudo ./install_dependencies.sh

3. Install and/or update dependencies.

Run the installation script, also in the scripts/ directory. 

$ sudo ./install_rtxi.sh

You will be prompted to select a configuration: 1 for Xenomai (real-time) or 2 for POSIX (non-real-time). Soon, RTXI will be compiled and installed. When finished, you can run v2.0 with: 

$ sudo rtxi

4. Install modules.

All available modules in 1.4 came packaged with RTXI. For 2.0, we have split modules into separate GitHub repositories so that users can install only what they need and also so that they can fork individual modules for customization, rather than having to fork all of RTXI.

What this means for you is that modules installed for 1.4 have not been updated for 2.0. You will have to update them manually. Go to our modules page to find and install the modules you need. If you're using your own custom modules, then they will need to be updated to use Qt4 instead of Qt3. Check out the Qt documentation for porting to Qt4 for help.
